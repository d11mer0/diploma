## 📖 Опис Проєкту

Цей проєкт є комплексною інформаційною системою, розробленою для автоматизації процесу аналізу освітніх програм (силабусів) на відповідність європейському стандарту IT-компетенцій **e-Competence Framework (e-CF)**.

Система використовує гібридний підхід на основі **штучного інтелекту (AI)** та **обробки природної мови (NLP)** для мапінгу текстових описів навчальних дисциплін до конкретних компетенцій та їх рівнів складності.

**Основні можливості:**
-   **Автоматичний аналіз:** Завантаження документів (.pdf, .docx) або вставка тексту для миттєвого аналізу.
-   **Інтерактивна візуалізація:** Наочні дашборди, включаючи "Індекс IT-релевантності", радарну діаграму покриття областей e-CF та розподіл за рівнями складності.
-   **Explainable AI (Пояснювальний ШІ):** Система не просто видає результат, а й показує, на основі яких ключових слів (технологій, дієслів Блума) було зроблено висновок.
-   **Експертна валідація:** Рольова модель дозволяє експертам перевіряти та підтверджувати результати роботи моделі.
-   **Active Learning (Активне Навчання):** Система збирає підтверджені експертами дані для подальшого донавчання (fine-tuning) ML-моделі, що з часом покращує її точність.

## 🚀 Технологічний Стек

Проєкт побудований на мікросервісній архітектурі та повністю докеризований для легкого розгортання.

-   **Фронтенд:**
    -   **Фреймворк:** React (Create React App)
    -   **Мова:** TypeScript
    -   **UI-бібліотека:** Material-UI (MUI) v5
    -   **Графіки:** Recharts
    -   **Веб-сервер:** Nginx (в Docker-контейнері)

-   **Бекенд (Gateway):**
    -   **Фреймворк:** NestJS
    -   **Мова:** TypeScript
    -   **Робота з БД:** Prisma ORM
    -   **Аутентифікація:** JWT (Access + Refresh Tokens)

-   **ML-сервіс:**
    -   **Фреймворк:** FastAPI
    -   **Мова:** Python
    -   **AI-моделі:**
        -   `all-MiniLM-L6-v2` (Sentence-Transformers Bi-Encoder)
        -   `cross-encoder/stsb-distilroberta-base` (Cross-Encoder)
    -   **NLP-бібліотека:** spaCy (для розпізнавання технологій та рівнів Блума)

-   **База даних:**
    -   **СУБД:** PostgreSQL

-   **Інфраструктура:**
    -   **Контейнеризація:** Docker, Docker Compose

## ⚙️ Запуск та Розгортання

Для запуску проєкту на локальній машині необхідно мати встановлений **Docker** та **Docker Compose**.

1.  **Клонуйте репозиторій (якщо потрібно):**
    ```sh
    git clone https://github.com/d11mer0/diploma.git
    cd diploma
    ```

2.  **Запустіть усі сервіси:**
    Виконайте одну команду в кореневій папці проєкту:
    ```sh
    docker-compose up --build -d
    ```
    -   `--build`: Примусово перезібрати образи, якщо були зміни в коді.
    -   `-d`: Запустити в "detached" режимі (у фоні).

    **Увага:** Перший запуск може зайняти 10-15 хвилин, оскільки Docker буде завантажувати базові образи та важкі ML-моделі.

3.  **Заповніть базу даних (лише при першому запуску):**
    Після того, як контейнери запустяться, виконайте команду для заповнення таблиці компетенцій:
    ```sh
    docker exec -it syllabus-backend sh -c "npx tsc prisma/seed.ts && node prisma/seed.js"
    ```

4.  **Відкрийте проєкт у браузері:**
    -   **Фронтенд:** [http://localhost:3001](http://localhost:3001)
    -   **Бекенд (API):** [http://localhost:3000/api](http://localhost:3000/api)
    -   **ML-сервіс:** [http://localhost:8000/docs](http://localhost:8000/docs)

## 🔧 Керування Базою Даних

Для ручного редагування даних (наприклад, для зміни ролі користувача на `ADMIN` або `EXPERT`) можна використовувати **Prisma Studio**.

1.  **Переконайтеся, що у файлі `.env` в папці `syllabus-gateway` вказано правильний `DATABASE_URL`:**
    ```
    DATABASE_URL="postgresql://root:rootpassword@localhost:5432/syllabus_db?schema=public"
    ```

2.  **Запустіть Prisma Studio:**
    У терміналі, в папці `syllabus-gateway`, виконайте:
    ```sh
    npx prisma studio
    ```
    Студія відкриється у вашому браузері.
